# PRD: Idle Factory

## 概要

ボクセルベースの3D工場建設ゲーム。機械を配置し、自動化ラインを構築して資源を加工・納品する。

## ターゲットユーザー

- 工場ゲーム好き（Factorio, Satisfactory, Mindustry経験者）
- カジュアルに遊びたい人（Factorioほど複雑じゃなくていい）
- 自動化・効率化に快感を覚える人

## コア体験

1. **配置の快感** - 機械を置いてラインが動き出す瞬間
2. **効率化の快感** - ボトルネックを発見し、改善する
3. **成長の実感** - 小さな手動採掘から巨大自動工場へ

## 成功指標

| 指標 | 目標 |
|------|------|
| 初回プレイ時間 | 5時間以上遊べる |
| チュートリアル完了率 | 90%以上 |
| メインクエスト完了率 | 50%以上 |

## v1.0 の定義

**「コンテンツ追加が作業になる状態」**

- 新機械追加 = game_specに20行 + モデル配置
- 新レシピ追加 = game_specに5行
- 新クエスト追加 = game_specに10行

## 非目標（やらないこと）

- モバイル対応（PC+Web優先）
- マルチプレイ（v1.0では）
- 複雑な電力・流体システム（v1.0では）
- Factorio級の複雑さ

## 技術制約

| 制約 | 理由 |
|------|------|
| Rust + Bevy | AIバイブコーディングに最適（型がAIを守る） |
| 全部コード | GUI操作不要でAIが完結できる |
| 依存最小 | Bevyバージョンアップのリスク軽減 |

## 差別化ポイント

- **ボクセル世界で工場建設** - Minecraftの見た目 + Factorioのゲーム性
- **シンプルさ** - 電力・流体なしでも成立
- **Idleゲーム要素** - 放置しても工場が動く

## 開発方針

このプロジェクトは「非エンジニアがAIだけで大規模ゲームを作れるか」の実験でもある。

判断に迷ったら：
1. **ゲームが遊べるようになるか？** → やる
2. **シンプルな方を選ぶ** → 複雑さは後から足せる
3. **既存パターンに従う** → 新しい設計は慎重に
