# Pattern Evaluation Skill

新機能や変更を評価するためのスキルです。

## 使用方法

機能実装後、このスキルを呼び出して評価を実行します。

---

## クイック評価チェックリスト

### 1. ECS設計（Bevy）

```
□ コンポーネントは単一責任か
□ 不要なコンポーネント追加/削除を避けているか
□ システム順序が明示されているか
□ Changed/Addedを活用しているか
□ 並列実行の機会を活かしているか
```

### 2. パフォーマンス

```
□ 60 FPS維持可能か
□ 描画コールは500以下か
□ インスタンシングを活用しているか
□ 不要なアロケーションを避けているか
□ シェーダースタッターがないか
```

### 3. UI/UX

```
□ 全操作に視覚フィードバックがあるか
□ ツールチップがあるか
□ Undo/Redoがあるか
□ キーボードショートカットが表示されているか
□ 情報階層が適切か
```

### 4. サウンド

```
□ ダイアログは常に明瞭か
□ 頻出音にバリエーションがあるか
□ カテゴリ別ボリューム調整可能か
□ 空間オーディオが適切か
```

### 5. アクセシビリティ

```
□ 色だけに頼っていないか
□ コントラスト比は4.5:1以上か
□ 字幕があるか
□ 全キーがリマップ可能か
```

### 6. セキュリティ（マルチプレイ）

```
□ サーバー権威か
□ 全入力をバリデーションしているか
□ レート制限があるか
□ セーブデータが暗号化されているか
```

### 7. ネットワーク（マルチプレイ）

```
□ クライアント予測があるか
□ 差分圧縮を使用しているか
□ 再接続処理があるか
□ 帯域は適切か
```

### 8. コード品質（Rust）

```
□ 不必要なcloneを避けているか
□ unwrap/expectを避けているか（初期化以外）
□ エラー型が定義されているか
□ テストがあるか
```
