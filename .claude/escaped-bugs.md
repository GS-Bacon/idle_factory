## 分析テンプレート

新しいバグを記録する際のチェックリスト:

1. **どの層で検出すべきだったか？**
   - [ ] ユニットテスト（関数単位のロジック）
   - [ ] E2Eテスト（システム連携）
   - [ ] VLMチェック（視覚的問題）
   - [ ] ファズテスト（境界値・異常入力）
   - [ ] proptestで（不変条件違反）

2. **なぜ検出できなかったか？**
   - [ ] テストケースが存在しない
   - [ ] テストケースの条件が不十分
   - [ ] テスト不可能（環境依存など）

3. **再発防止策**
   - [ ] テスト追加
   - [ ] VLMプロンプト改善
   - [ ] コードレビュー観点追加
