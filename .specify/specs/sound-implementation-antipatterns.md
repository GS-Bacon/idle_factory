# ゲームサウンド実装アンチパターン

**作成日**: 2025-12-22
**目的**: プレイヤーを不快にさせるオーディオ実装の失敗例と対策

---

## 1. 致命的アンチパターン

### 1.1 ダイアログ埋没（Dialog Drowning）

> 「Mass Effectは最悪の例。群衆のノイズがメインキャラクターの台詞より大きい」
> — ResetEra プレイヤーレビュー

**症状**:
- ボイスがSE/BGMにかき消される
- 音量を上げてもセリフが聞こえない
- 字幕なしではストーリーが追えない

**実例**:
- Mass Effect: 環境音がダイアログを圧倒
- Last of Us 2: 一部のセリフが「ほぼ聞こえないほど小声」
- Ghost of Tsushima: 背景会話が調整スライダーで改善しない

**対策**:
```
✓ ダイアログ再生時に他音声をダッキング（10-15dB）
✓ ダイアログ専用スライダー必須
✓ ダイアログは-24 LUFSでマスタリング
✓ 混雑シーンでは環境音を自動減衰
```

---

### 1.2 爆発音ジェットコースター（Explosion Roller Coaster）

**症状**:
- 通常時は適切な音量
- 爆発/大音量イベントで耳をつんざく
- プレイヤーが常にボリューム調整を強いられる

**実例**:
- 多くのFPSゲーム
- アクションゲームの大規模戦闘シーン

**対策**:
```
✓ リミッター/コンプレッサーを最終段に適用
✓ ダイナミックレンジを適切に制限（-12dB以内）
✓ 大音量イベントは自動ダッキング
✓ ヘッドルームを確保した設計
```

---

### 1.3 空間オーディオ崩壊（Spatial Audio Breakdown）

> 「Dishonored 2では隣の部屋の敵が目の前にいるかのように聞こえる。ステルスゲームでこれは致命的」
> — ResetEra プレイヤーレビュー

**症状**:
- 音の方向がわからない
- 距離感が狂っている
- 左右/前後が逆転

**実例**:
- The Forest: 「位置オーディオの概念がない」
- Dishonored 2: 距離による減衰が機能しない

**対策**:
```
✓ 距離減衰カーブを適切に設定
✓ 壁/障害物による遮蔽を実装
✓ ローパスフィルターで距離感を表現
✓ テストプレイでヘッドフォン確認
```

---

### 1.4 反復地獄（Repetition Hell）

> 「Borderlands 2のBane武器は弾と同じ速さで叫ぶ。正気を削る」
> — TV Tropes

**症状**:
- 同じ音が延々と繰り返される
- プレイヤーがサウンドをオフにしたくなる
- 長時間プレイで疲労が蓄積

**実例**:
- Navi (ゼルダ): 「HEY!!! LISTEN!!!」の連呼
- ソニック: ジャンプ音が最も聞く音なのに1種類
- RPGの宝箱: 100個開けても同じ音

**対策**:
```
✓ 頻出音は3-5種類のバリエーション
✓ ピッチ/ボリュームのランダム化（±5-15%）
✓ 「迷惑スコアシステム」で問題音を追跡
✓ 連続再生時は間引き
```

## 2. ミキシングアンチパターン

### 2.1 スライダー不足（Slider Starvation）

**症状**:
- マスターボリュームのみ
- カテゴリ別調整不可
- プレイヤーの好みに対応できない

**最低限必須**:
```
□ マスター
□ BGM
□ SE
□ ボイス
```

**推奨追加**:
```
□ 環境音
□ UI音
□ 通知音
```

---

### 2.2 帯域衝突（Frequency Collision）

**症状**:
- BGMとSEが同じ周波数帯で喧嘩
- 重要な音が埋もれる
- 全体的に濁った音質

**実例**:
- BGMのベースと爆発音が重なる
- UIクリック音とゲームSEが同じ帯域

**対策**:
```
✓ BGMとSEで使用帯域を分離
✓ イコライザーで周波数住み分け
✓ サイドチェイン圧縮で自動回避
✓ 作曲時からSEスペースを確保
```

---

### 2.3 圧縮アーティファクト（Compression Artifacts）

**症状**:
- 耳障りなノイズ
- 高音がシャリシャリ
- 長時間聴くと疲れる

> 「圧縮アーティファクトは『耳の疲労』を引き起こし、脳が無意識に音を解読しようとして精神的に疲弊する」

**対策**:
```
✓ 十分なビットレートを確保（MP3なら192kbps以上）
✓ OGG Vorbis推奨（同ビットレートで高品質）
✓ ループ音は特に高品質で
✓ 圧縮前にノーマライズ
```

## 3. UIオーディオアンチパターン

### 3.1 シャープアタック（Sharp Attack）

**症状**:
- ボタンクリックが耳に刺さる
- 突然の大きな音で驚く
- 音量を下げたくなる

**対策**:
```
✓ 全UI音に10ms以上のフェードイン
✓ トランジェントを滑らかに
✓ ピーク音量を制限
```

---

### 3.2 過剰通知（Notification Overload）

**症状**:
- 些細なイベントまで音が鳴る
- 通知音が途切れない
- プレイに集中できない

**実例**:
- テキストチャット全メッセージに通知音
- インベントリの些細な変更まで音

**対策**:
```
✓ 重要度に応じて通知を階層化
✓ 頻繁なイベントは音を省略
✓ 通知音のクールダウン設定
✓ カテゴリ別オン/オフ設定
```

---

### 3.3 感情的混乱（Emotional Confusion）

**症状**:
- 成功音が暗く聞こえる
- エラー音が楽しそう
- 意味と音が一致しない

**正しい対応**:
| 意味 | 音の特性 |
|------|----------|
| 成功 | 上昇音、協和音、明るい |
| 失敗 | 下降音、不協和音、暗い |
| 確認 | 短く、はっきり |
| キャンセル | やや長く、落ち着く |

---

## 4. 技術的アンチパターン

### 4.1 無制限同時発音（Unlimited Polyphony）

**症状**:
- 100台の機械が同時に鳴る
- CPU使用率が急上昇
- 音割れ・クラッシュ

**対策**:
```
✓ 最大同時発音数を設定（32-64推奨）
✓ 優先度システムで重要音を保護
✓ 遠距離音は間引き
✓ 同種音のインスタンス制限
```

---

### 4.2 ハードコードパス（Hardcoded Audio Paths）

**症状**:
- 環境によって音が再生されない
- MOD対応が困難
- 音声ファイル変更に再コンパイル必要

**対策**:
```
✓ アセット参照はデータドリブンに
✓ パスは設定ファイルで管理
✓ ホットリロード対応
```

---

### 4.3 未処理の同期（Unsynced Animation）

> 「機械の音をアニメーションに同期させることは、最も要望の多かった機能の一つ」
> — Factorio開発者ブログ

**症状**:
- 動きと音がずれる
- 違和感・没入感低下

**対策**:
```
✓ アニメーションイベントで音をトリガー
✓ Sound Accents（特定フレームで短い音）
✓ ループ音の開始/終了をアニメに合わせる
```

---

## 5. アクセシビリティアンチパターン

### 5.1 音声のみ情報（Audio-Only Information）

**症状**:
- 重要情報が音声のみ
- 字幕なし
- 聴覚障害者がプレイ不可

**対策**:
```
✓ 全ダイアログに字幕
✓ 効果音にも字幕オプション [爆発音] [足音]
✓ 重要音声は視覚エフェクトも併用
```

---

### 5.2 視覚障害者無視（Blind Player Neglect）

**症状**:
- UIが音声読み上げ非対応
- 空間的手がかりがない
- ゲーム進行に視覚必須

**対策**:
```
✓ メニュー音声読み上げ
✓ 重要オブジェクトに音響定位
✓ ソナー/Ping機能
```

---

## 6. 工場ゲーム特有のアンチパターン

### 6.1 機械カオス（Machine Chaos）

**症状**:
- 50台の機械が同じ音で鳴る
- どの機械が問題かわからない
- 騒音で疲弊

**対策**:
```
✓ 機械タイプ別に異なる音色
✓ 距離による減衰を強めに
✓ カメラズームでアンビエント切り替え
✓ 「機械音」専用スライダー
```

---

### 6.2 警告音疲労（Alert Fatigue）

**症状**:
- 電力不足警告が鳴り続ける
- ライン停止通知が連発
- 警告を無視するようになる

**対策**:
```
✓ 警告音にクールダウン（30秒等）
✓ 同種警告は統合
✓ 緊急度で音を変える
✓ 視覚表示との併用
```

---

### 6.3 進捗音過剰（Progress Sound Spam）

**症状**:
- アイテム生産ごとに音
- 1秒に何十個も生産→音が途切れない

**対策**:
```
✓ 高速生産時は音を間引き
✓ バッチ完了時のみ音
✓ アイテム落下音を減衰
```

---

## 7. アンチパターン速見表

### 致命的（必ず回避）

| パターン | 症状 | 対策 |
|----------|------|------|
| ダイアログ埋没 | セリフが聞こえない | ダッキング必須 |
| 爆発音問題 | 耳をつんざく | リミッター |
| 空間崩壊 | 方向がわからない | 減衰カーブ調整 |
| 反復地獄 | 同じ音が延々 | バリエーション化 |

### 強く回避

| パターン | 症状 | 対策 |
|----------|------|------|
| スライダー不足 | 調整不可 | カテゴリ別スライダー |
| 帯域衝突 | 音が濁る | 周波数分離 |
| シャープアタック | 耳に刺さる | フェードイン |
| 過剰通知 | 集中できない | 階層化・省略 |

### 注意

| パターン | 症状 | 対策 |
|----------|------|------|
| 無制限発音 | CPU負荷 | 上限設定 |
| 未同期 | ずれ | アニメイベント |
| 機械カオス | 騒音 | 距離減衰強化 |
| 警告疲労 | 無視される | クールダウン |

## 8. チェックリスト

### ミキシング
- [ ] ダイアログが常に明瞭か
- [ ] 爆発音で耳が痛くならないか
- [ ] 音量スライダーは十分か
- [ ] 帯域の衝突はないか

### 反復
- [ ] 頻出音にバリエーションがあるか
- [ ] ピッチ/ボリュームはランダム化されているか
- [ ] 1時間プレイして疲れないか

### 空間
- [ ] 音の方向がわかるか
- [ ] 距離による減衰は自然か
- [ ] 壁による遮蔽があるか

### アクセシビリティ
- [ ] 字幕オプションがあるか
- [ ] 視覚のみでプレイ可能か
- [ ] 聴覚のみでプレイ可能か
